Groupes de textures Le gestionnaire de groupes de textures est disponible dans le menu Outils de l'IDE. Ici, vous pouvez ajouter, supprimer et renommer des groupes de textures. GameMaker Studio 2 vous permet d'affecter chacune des ressources graphiques (sprites, jeux de tuiles et polices) à différents groupes de textures afin d'optimiser le nombre de changements de textures effectués par le matériel à un moment donné dans votre jeu. de changements de texture effectués par le matériel à un moment donné dans votre jeu. Pour que cela fonctionne, vous devez définir les groupes de texture nécessaires à partir de cette fenêtre, puis passer par chacun des types d'actifs appropriés et sélectionner le groupe auquel vous souhaitez qu'il appartienne dans le menu déroulant. groupe auquel vous souhaitez qu'il appartienne dans le menu déroulant. Vous pouvez également ajouter des ressources à un groupe de texture en cliquant avec le bouton droit de la souris sur le menu dans le navigateur de ressources. Vous pouvez sélectionner un ou plusieurs actifs (ou un dossier entier de groupe d'actifs), puis utiliser le menu RMB pour ajouter les actifs à n'importe quel groupe, comme illustré ci-dessous pour les jeux de tuiles : il est important de réaliser qu'un groupe de texture unique peut avoir plusieurs pages de texture, et qu'elles ne sont pas équivalentes. Si votre jeu comporte beaucoup de graphiques uniques pour chaque niveau, vous pouvez définir un seul groupe de texture pour chacun des niveaux, mais les graphiques nécessitent deux pages de texture ou plus par groupe. Par défaut, toutes les images de votre jeu seront ajoutées au groupe de textures "Default", qui existe toujours. Cependant, vous pouvez cliquer sur le bouton Ajouter nouveau pour créer un nouveau groupe de texture, et si vous sélectionnez le nom du groupe, vous pouvez Si vous sélectionnez le nom du groupe, vous pouvez le modifier de manière appropriée. Pour commencer, le groupe affichera " cegroupe est vide " pour chaque section du groupe (Sprites, tilesets et polices), et vous devrez donc passer en revue vos ressources graphiques et les ajouter. Vous pouvez ajouter une ressource en utilisant le bouton Ajouter une ressource en bas, ou en sélectionnant le groupe de texture dans l'éditeur de ressources pour l'image, le jeu de tuiles ou la police. Les groupes de texture autres que le groupe "Default" peuvent être supprimés à tout moment. (notez que si vous supprimez un groupe par erreur, vous pouvez utiliser / + "Z" pour annuler l'action). Une fois que vous avez défini les groupes de textures - et que vous leur avez attribué des ressources graphiques - vous les verrez listés à gauche de la fenêtre principale de l'éditeur de groupes de textures. à droite. Ceci est particulièrement utile lorsque vous travaillez sur de grands projets et que vous devez être en mesure de voir quelles images ont été assignées à quel groupe. Notez que si les images assignées au groupe ont plusieurs sous-images (c'est-à-dire qu'elles sont animées), alors seule la première sous-image est affichée dans la fenêtre d'aperçu. Vous pouvez également cliquer avec le bouton droit de la souris sur une ressource pour ouvrir un petit menu vous donnant l'option de de la déplacer vers un autre groupe de textures ou d'ouvrir l'éditeur de cette ressource. Il convient de noter que, même si les ressources sont affichées dans la liste comme étant séparées, tous les sprites, tilesets et polices seront ajoutés à la même page (ou pages) de texture, et non à une page individuelle pour chaque type de ressource. La seule exception à cette règle La seule exception à cette règle est lorsqu'un sprite a été marqué comme étant pour une "page de texture séparée", dans ce cas, il recevra sa propre page de texture individuelle (voir la section sur les "paramètres de texture" dans l'éditeur de sprites pour plus de détails). Pour plus d'informations sur les pages de texture et la manière dont elles sont générées, veuillez consulter la page suivante : Pages de texture Les paramètres de groupe dans l'éditeur de groupe de texture sont utilisés pour déterminer exactement comment les pages de texture pour un groupe donné doivent être créées, avec les options suivantes disponibles : Allow Scaling (Autoriser la mise à l'échelle) - En désactivant cette option, vous obligerez GameMaker Studio 2 à ne pas mettre à l'échelle le groupe de textures, quelles que soient les circonstances. Cela peut s'avérer utile, notamment dans le cas d'applications universelles, car cela vous permet de placer des ressources spécifiques telles que les polices de caractères les polices dans un groupe à part entière qui sera utilisé lorsque l'application est exécutée sur un périphérique spécifique. Par exemple, les polices peuvent être superbes sur un écran rétina, mais lorsqu'elles sont mises à l'échelle pour être dessinées sur un écran normal, elles peuvent être terribles. groupe spécial sans mise à l'échelle - si vous développez pour l'iPhone, vous souhaiterez peut-être vérifier cette option, car des applications ont été rejetées parce que les images étaient floues en raison de la mise à l'échelle de la texture. Cette option est activée par défaut. Recadrage automatique - Lorsque vous ajoutez des images dans GameMaker Studio 2, si elles ont de l'"espace" autour d'elles (c'est-à-dire une transparence de 100 %), elles sont automatiquement recadrées lorsqu'elles sont ajoutées à la page de texture et l'option "Recadrage automatique" est cochée. Automatiquement" est cochée. Cependant, si, par exemple, vous utilisez l'image comme une texture non répétitive pour des primitives ou quelque chose de similaire, cela signifie que les coordonnées UV pour le mappage de texture ne seront pas correctement représentées. Dans ce cas vous pouvez désactiver cette option et le sprite sera ajouté à la page de texture "tel quel", en conservant toutes les transparences qu'il peut avoir sur les bords. Cette option est activée par défaut. Generate Mipmaps - En activant cette option (elle est désactivée par défaut), vous indiquez à GameMaker Studio 2 que la page de texture originale doit être mise à l'échelle et filtrée en plusieurs résolutions dans le fichier de texture. Cette option est normalement désactivée, mais si vous utilisez des caméras pour effectuer un zoom avant ou arrière sur de grandes zones de la salle de jeu, ou si vous travaillez avec un jeu en 3D, vous pouvez l'activer et utiliser les fonctions Mipmapping pour pour régler la façon dont le jeu traite les différents niveaux de mipmap. Lorsque vous cochez cette case, la valeur "Border Size" (taille de la bordure) sera fixée à 8 (si elle était fixée à moins de 8 à l'origine). Cela est dû au fait que, lorsque vous utilisez des mipmaps, vous avez besoin de bordures plus grandes entre les images sur les pages de texture, sinon elles commencent à s'effilocher. images sur les pages de texture, sinon elles commencent à se mélanger à des niveaux de mipmap plus élevés. La valeur par défaut de 8 permet de réduire l'image à environ un huitième de sa taille d'origine avant que les sous-images ne commencent à se mélanger. mais vous pouvez modifier cette valeur pour qu'elle corresponde à vos besoins. Notez que si vous utilisez l'option "SeparateTexture Page" pour les sprites, vous n'avez pas besoin de passer par chaque texture ici et de la marquer pour générer des mipmaps, mais vous pouvez aller dans les Options générales du jeu et activer l'option "Générer desmipmaps pour les pages de textures séparées". Taille de la bordure - La taille de la bordure modifie la quantité de chevauchement (ou "doublage des pixels") autour des bords de toutes les ressources sur les pages générées pour le groupe de textures, de sorte que lorsque les images sont mises à l'échelle, aucun vide (ou couture) n'est visible entre elles. visibles entre elles. La valeur par défaut est de 2 pixels, mais dans les cas extrêmes où vous effectuez un zoom avant ou arrière important, vous devrez peut-être augmenter cette valeur. Notez que l'éditeur de tuiles dispose une option pour définir la bordure de sortie pour les tuiles (et fournit également une explication de la raison pour laquelle cela est nécessaire), et ceci est distinct du réglage de la bordure ici - fondamentalement, les tuiles sont générées avec une bordure de sortie pour chaque tuile sur une texture temporaire, et celle-ci est ensuite ajoutée à la page de texture avec la bordure donnée ici autour du bord extérieur. Notez également que l'activation des Mipmaps définira automatiquement la taille de la bordure par défaut pour la page de texture à 8 pour aider à prévenir le "saignement" entre les textures à des niveaux de Mip plus élevés. textures à des niveaux de Mip plus élevés, mais ceci peut être changé. Groupe de texture parent - Ici, vous pouvez choisir n'importe quel groupe existant pour qu'il devienne le groupe "parent" de celui qui est actuellement sélectionné. pour la gestion des ressources lorsque vous travaillez avec des jeux multiplateformes. Un groupe de textures avec un parent est toujours un groupe de textures à part entière et génère des pages de textures de la même manière que n'importe quel autre groupe, mais le fait de leur attribuer un parent vous permet d'exclure de grandes parties pour différentes plates-formes. Par exemple, vous pouvez ajouter un grand nombre de sprites de joueurs Android à un groupe "player", et des sprites d'ennemis Android à un groupe "enemy", puis leur faire utiliser un groupe parent "android". Si vous excluez ensuite le groupe parent "android" de la compilation pour Windows ou une autre plate-forme, les groupes enfants seront également exclus. Outre ces paramètres, vous pouvez également choisir d'exporter (ou non) un groupe de textures donné vers une plate-forme donnée en cochant ou décochant les cases appropriées dans la section Utiliser le groupe sur... de la fenêtre. Notez cependant que le groupe de textures est toujours exporté vers toutes les plates-formes, et vous ne pouvez pas cocher/décocher l'une d'entre elles (seuls les groupes de textures personnalisés le permettent). Les groupes de textures sont également liés aux paramètres des configurations. Cela signifie que vous pouvez définir des groupes de textures pour chaque configuration individuelle de votre jeu, puis attribuer les ressources sprites, jeux de tuiles ou polices à ces différents groupes. à ces différents groupes et cela sera "mémorisé" lorsque vous passerez à une autre configuration. Il en va de même pour toutes les options d'exportation que vous avez choisies dans la fenêtre Groupe de textures. à exporter vers des plates-formes spécifiques, et ce, pour chaque configuration. Vous pouvez, par exemple, créer une configuration iPad et créer des pages de texture de 2048x2048px (configuration via les options du jeu). Vous attribuez ensuite toutes les ressources graphiques au groupe de texture approprié. et, une fois terminé, vous pouvez passer à une autre configuration (pour HTML5, par exemple) et, à partir de la fenêtre Groupe de texture, créer de nouveaux groupes de texture de 512x512px, en affectant les ressources à ces nouveaux groupes. Il existe également un bouton en bas de la page qui vous permet de copier les paramètres du groupe de texture d'une configuration à une autre. Enfin, les groupes de texture ne concernent que les images bitmap, et les sprites non bitmap ne seront pas affectés à des groupes de texture.
