part_system_automatic_update Cette fonction contrôle si GameMaker Studio 2 doit mettre à jour le système de particules automatiquement ou non, et normalement vous n'auriez pas besoin d'utiliser cette fonction comme valeur par défaut de true (la mise à jour automatique est activée) est ce que vous souhaitez. Cependant, pour les effets spéciaux ou pour mettre le jeu en pause, vous pouvez le régler sur false et cela empêchera toute mise à jour effectuée sur le système de particules donné à moins que cela ne soit explicitement commandé par l'utilisation de la fonction part_system_update() ou vous utilisez à nouveau cette fonction pour activer les mises à jour automatiques. Aucune mise à jour signifie que les particules créées dans ce système ne changeront plus de position, de couleur ou de tout autre paramètre et que les émetteurs cesseront également de fonctionner, "gelant" le système au point exact où la mise à jour automatique a été définie sur false. Une chose à noter est que si vous utilisez les effets simples créés par les fonctions effect_create_above() ou effect_create_below() alors vous pouvez utiliser les valeurs de 0 (pour les effets ci-dessous) ou 1 (pour les effets ci-dessus) comme index du système de particules et ainsi basculer la mise à jour automatique pour ceux-ci également (cela fonctionnera également pour activer la mise à jour des DnD de particules DnD™). Syntaxe:part_system_automatic_update(ind, automatic); Argument Description ind L'indice du système de particules à changer. automatic Si la mise à jour automatique est activée (true ) ou non (faux). Retour:N/A Exemple:if global.Pause     {     part_system_automatic_update(global.Sname, false);     } Le code ci-dessus désactivera les mises à jour des particules si la variable globale "Paused" est vraie.
