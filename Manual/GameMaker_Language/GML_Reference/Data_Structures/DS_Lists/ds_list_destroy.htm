ds_list_destroy

这个函数将从内存中删除给定的列表数据结构，释放它所使用的资源并删除它所包含的所有值。这个函数应该总是在你使用完ds_list时使用，以防止内存泄漏 以防止内存泄漏，因为内存泄漏会使你的游戏变慢或崩溃。注意，销毁一个列表将取消对其中存储的任何数据结构的引用，从而导致内存泄漏，所以你需要在销毁列表之前手动销毁所有数据结构项 来防止这种情况。唯一不需要这样做的时候是当你把列表中的任何项目标记为另一个DS列表或DS地图时，在这种情况下这些项目将被销毁，它们的 内存也会被自动清理掉。

重要的是!当你创建一个数据结构时，识别它的索引值是一个从0开始的整数值。 这意味着不同类型的数据结构可以有相同的索引值，所以如果有疑问，你应该使用 ds_exists()函数来访问它们。还要注意的是，索引是可以重复使用的，所以一个被破坏的数据结构索引值可能会被之后新创建的数据结构使用，所以我们建议 所以我们建议在销毁数据结构后，总是将持有DS索引的变量设置为-1。

语法。

ds_list_destroy(id)。

争论 描述
身份证 要删除的数据结构的ID。

返回。

不适用

例子。

如果（lives == 0）
{
ds_list_destroy(AI_list);
AI_list = -1;
room_goto(rm_Menu);
}

上面的代码将检查内置的全局变量 "lives "的值，如果它是0，它将破坏变量 "AI_list "中索引的DS列表，然后改变房间。

返回。DS列表
