ds_grid_destroy

这个函数将从内存中删除给定的网格数据结构，释放它所使用的资源，并删除它所包含的所有值。这个函数应该在你结束使用DS网格的时候使用，以防止内存泄漏 以防止内存泄漏，因为内存泄漏会使你的游戏变慢或崩溃。

重要的是!当你创建一个数据结构时，识别它的索引值是一个从0开始的整数值。 这意味着不同类型的数据结构可以有相同的索引值，所以如果有疑问，你应该使用 ds_exists()函数来访问它们。还要注意的是，索引是可以重复使用的，所以一个被破坏的数据结构索引值可能会被之后新创建的数据结构使用，所以我们建议 所以我们建议在销毁数据结构后，总是将持有DS索引的变量设置为-1。

语法。

ds_grid_destroy(index)。

争论 描述
指数 这个要销毁的网格的索引。

返回。

不适用

例子。

if (lives == 0)
{
ds_grid_destroy(Wall_Grid);
Wall_Grid = -1;
room_goto(rm_Menu);
}

上面的代码将检查内置的全局变量 "lives "的值，如果它是0，它将破坏变量 "Wall_Grid "中索引的DS网格，然后改变房间。
