show_debug_overlay

在测试你的游戏时，这个功能可以用来打开和关闭标准的调试覆盖层，默认情况下是禁用的。调试覆盖会在实际的游戏窗口中显示一个CPU/GPU使用率的图形条，以及当前的真实帧值。 纹理交换的数量和顶点批处理的数量（注意，纹理交换和顶点批处理永远不会是零，通常会显示2或3的值，因为即使是在一个空的房间和没有对象的情况下，GameMaker Studio 2仍然需要绘制和批处理 的事情）。)

这个条形图被分成若干部分，每部分是1/60秒。从上面的图片可以看出，这个条形图是由不同的 颜色。

  • 绿色- 输入/输出处理（即：键盘、鼠标、游戏手柄、网络等）。
  • 红色- 阶梯事件的更新速度
  • 黄色- 抽奖活动所需的时间
  • 橙色-调试更新时间，通常只有在使用调试模块时才能看到。
  • 白色- GPU剩余时间，即在下一帧开始之前等待GPU完成渲染的时间。
  • Cyan- 文本渲染时间
  • 灰色- 清理屏幕的每一个抽奖步骤所需的时间
  • Blue-垃圾收集器运行所需的时间
  • 暗红色- GPU冲洗，即GPU需要多长时间从内存中清除图像。

使用这个功能，无论是否在调试模式下，你都可以添加覆盖层，这样，你就可以看到你的游戏是如何有效地运行的，并得到一个关于它是如何使用可用资源的视觉提示，而没有调试器运行的开销。 旁边。

注意：由于HTML5目标的工作方式与所有其他目标不同，该功能在HTML5目标上将不起作用。

语法。

show_debug_overlay（启用）。

争论 描述
启用 开启（真）或关闭（假）调试覆盖。

返回。

不适用

例子。

如果 global.debug
{
show_debug_overlay(true);
}
else
{
show_debug_overlay(false);
}

上面的代码将根据一个全局变量的值来切换调试的开启或关闭。
