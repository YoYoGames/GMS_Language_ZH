音频同步组_调试

这个函数可以用来显示任何给定同步组的调试信息（组的索引是使用函数创建组时返回的值 audio_create_sync_group()).你可以用-1的值调用该函数来关闭覆盖。 可以用-1的值调用该函数来关闭叠加。

当你在调试音频同步组时，覆盖层会显示。

  • 哪个同步组正在调试
  • 里面有多少个频道
  • 同步组有多少次失败（这意味着它试图做太多的工作）。
  • 频道列表
    • 红色表示这个频道目前没有做任何工作
    • 黄色表示该通道处于初始状态（尚未解码或播放）。
    • 绿色表示该通道目前正在寻找/解码
    • 紫色表示该通道的数据已经用完
  • 一个显示解码线程正在做什么工作的剖析条。
    • 蓝色表示正在首次读取一个文件
    • 红色表示线程正在读取一个文件，以确定从哪里读取。
    • 绿色表示线程正在对数据进行解码
    • 紫色表示该线程正在睡觉，直到需要工作为止

请注意，对于一般的音频调试，你还有以下函数 audio_debug().

注意：此功能不适用于HTML5目标平台。

语法。

audio_sync_group_debug(group_index)。

争论 描述
组_索引 要调试的组索引，或者-1表示关闭调试。

返回。

不适用

例子。

如果（debug_mode）
{
audio_sync_group_debug(sg);
}
else
{
audio_sync_group_debug(-1);
}

上述代码将打开或关闭音频同步组调试覆盖，显示变量 "sg "中索引的同步组，这取决于游戏是否在调试模式下运行。
