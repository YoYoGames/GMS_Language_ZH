keyboard_set_map

有时在制作游戏时，你可能希望一个键的作用与另一个键相同。例如，许多人使用WASD键进行移动，但也有许多人使用方向键！那么，该怎么办？那么，该怎么做呢？好吧，你可以对运动系统进行编码 两次，但这有点复杂，幸好是多余的，因为这个功能允许你将一个键 "映射 "到另一个键上，所以任何一个键的输入都会被解释为相同的。要做到这一点，你要选择一个你想映射的键（key2 - 这将是你编写代码的那个键）和一个你希望它被映射到的键（key1）。之后，GameMaker Studio 2将把对任何一个键的按压都解释为来自键2。你也可以用这个功能来设计 一个系统，用户可以通过简单地将用户输入键映射到你已经编码到游戏中的键来定义自己的游戏键。

该函数接受一个由函数返回的键码值 ord()(只有A-Z的大写字母或0-9的数字)，或主键盘输入页面上列出的任何vk_*常数。

重要的是!一旦你使用这个功能，你所映射的键（key1）将不再能作为它的实际按键符号使用。例如，如果你把向上箭头映射到 "W"，那么你将不再能够检测到按下 "W "键是什么意思。 W "键时，你将无法检测到它是一个 "W"，它将永远被认为是一个向上的箭头。要撤销这一点，要么将该键映射到它本身--因此键1和键2都是 ord("W") ，要么使用函数 keyboard_unset_map() 。

语法。

keyboard_set_map(key1, key2)。

争论 描述
钥匙1 这是键1要被映射到的钥匙。
钥匙2 这是要映射的钥匙

返回。

布尔型

例子。

keyboard_set_map(ord("A"), vk_left);

上面的示例代码将把 "A "键映射到左箭头键上。这意味着玩家可以使用 "A "键或左键，所有为左键编写的代码也将响应 "A" 键来代替。
