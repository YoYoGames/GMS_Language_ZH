绩效_负载_朋友

这个函数将向服务器发送一个请求，以获取所有登录用户的朋友的信息，并将触发一个回调的Social Asynchronous Event，其中包括 的 async_load地图填充了相关的键/值对。这个DS地图的id键被用来识别正确的回调（可以有一个以上的触发函数）。 对于任何给定的异步事件，可以有一个以上的触发函数），并将与常数 achievement_friends_info以及每个朋友的一些其他键/值对配对。该地图的具体内容是 如下所示。

注意：此功能仅适用于 iOS仅限。

  • " id" - 对于这个函数，它应该是achievement_friends_info
  • "FriendN" - 朋友的名字，其中 "N "是一个整数值，对应于他们在朋友列表中的位置。
  • "FriendidN" - 朋友的唯一用户ID，"N"。

扩展实例。

下面的代码可能会在玩家登录到他们的游戏账户后被调用，使用 achievement_login()来获取该用户所有的朋友列表。

achievement_friends_info()。

这将发送一个关于用户朋友信息的请求，并生成一个异步回调，其中包含以下数据的特殊async_loadDS map。

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_friends_info
{
var numfriends = ds_map_find_value(async_load, "numfriends");
global.numfriends = numfriends;
for(var i=0; i < numfriends; i++;)
{
global.friendname[i] = ds_map_find_value(async_load, "Friend" + string(i));
global.friendid[i] = ds_map_find_value(async_load, "Friendid" + string(i));
achievement_get_pic(global.friendid[i]) 。
}
}

上面的代码在社交异步事件中检查返回的DS地图，如果它的 "id "与被检查的常数相匹配，它就会在地图中循环，将所有不同的值存储在一些数组中，并请求 地图中每个条目的图像（见 achievement_get_pic()).
