ds_set_precision

当比较数值时，例如在地图中搜索或对列表进行排序时，GameMaker Studio 2必须决定两个数值是否相等。对于字符串和整数值来说，这是很清楚的，但是对于实数来说，由于浮动 点舍入误差，看似相等的数字很容易变成不相等。例如，(5/3)*3有可能不等于5!为了帮助避免这种情况，在所有实数函数上都使用了一个精度值 被用于所有的实数函数，当两个数字之间的差值小于这个精度时，它们被认为是相等的。所有的数据结构函数默认使用0.0000001的精度。 除非被这个函数改变。

注意：这个精度在所有的数据结构中都有使用，但在GML的其他比较中没有使用!

语法。

ds_set_precision(prec)。

争论 描述
沉淀 精度值（默认为0.0000001）。

返回。

返回:不适用

例子。

ds_set_precision(0.0001)。

上述代码将改变所有数据结构函数的默认精度设置。
