DS列表

DS列表是一种数据结构，它在添加信息时按顺序存储（很像一个数组）。DS 列表是非常灵活的数据结构，允许你在列表的末尾添加数值或在列表的中间插入数值。 列表中间的某个地方，以及给你洗牌的能力（使数值随机化）或以升序或降序排序。

当访问DS列表数据结构时，你应该总是尽量使用整数值来表示列表位置，如果你不这样做，所有非整数的索引都会被GameMaker Studio 2浮动。如果这不是你所需要的，那么你将需要 在传递你想检查的索引之前，你需要自己进行四舍五入。

注意：与所有的动态资源一样，数据结构会占用内存，所以当不再需要时，应该总是销毁，以防止内存泄漏，这将减缓并最终使你的游戏崩溃。

注意：如果你需要检查一个数据结构是否存在，你可以使用 ds_exists()函数。

除了下面列出的特定函数外，你都可以使用一个表达式（称为访问器）来添加或修改你的DS列表的内容。这个访问器看起来类似于一个1D数组，语法如下。

list_index[| index]

你可以从GML概述的Accessors页面中找到更多的信息和例子。

  • ds_list_find_value
  • ds_list_insert
  • ds_list_replace
  • ds_list_shuffle
  • ds_list_sort
  • ds_list_copy

Ds列表也可以用来保存DS地图和其他DS列表，因此我们有一些特殊的函数用来 "标记 "或 "标注 "列表中保存这些数据结构的位置。一般来说 一般来说，这不是必须的，但它的好处是当父列表被销毁时，可以自动收集（销毁）被标记的数据结构，而且在与 json_encode() 函数一起使用时，它也是必需的，以便对列表和它的数据进行正确的编码。这方面的函数有：。

  • ds_list_mark_as_list
  • ds_list_mark_as_map
  • ds_list_is_list
  • ds_list_is_map
