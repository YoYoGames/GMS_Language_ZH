steam_ugc_create_query_user

该函数可用于查询UGC数据库。该函数自动使用应用程序、用户的默认ID，并假定查询是由消费者（而不是创建者）进行的。该函数要求你使用以下 常量来查询数据的类型，查询项目的类型和返回项目的排序顺序，以及一个页码--注意，一个查询最多会返回50个项目。

列表类型常数 描述
ugc_list_Published 见Steam文档。
ugc_list_VotedOn -
ugc_list_VotedUp -
ugc_list_VotedDown -
ugc_list_WillVoteLater -
ugc_list_Favorited -
ugc_list_subscribed -
ugc_list_UsedOrPlayed -
ugc_list_Followed -

匹配类型常数 描述
ugc_match_Items 见Steam文档。
ugc_match_Items_Mtx -
ugc_match_Items_ReadyToUse -
ugc_match_Collections -
ugc_match_Artwork -
ugc_match_Videos -
ugc_match_屏幕截图 -
ugc_match_AllGuides -
ugc_match_WebGuides -
ugc_match_IntegratedGuides -
ugc_match_UsableInGame -
ugc_match_ControllerBindings -

排序顺序常数 描述
ugc_sortorder_CreationOrderDesc 见Steam文档。
ugc_sortorder_CreationOrderAsc -
ugc_sortorder_TitleAsc -
ugc_sortorder_LastUpdatedDesc -
ugc_sortorder_SubscriptionDateDesc -
ugc_sortorder_VoteScoreDesc -
ugc_sortorder_ForModeration -


该函数返回一个唯一的查询句柄值，该句柄值应存储在一个变量中，以便在其他查询函数中使用。请注意，这个函数只是准备查询，而不是实际发送 - 为此，你必须调用函数 steam_ugc_send_query.htm">steam_ugc_send_query()- 而且你可以使用进一步的steam_ugc_query_*()函数在实际发送之前完善搜索请求。

语法。

steam_ugc_create_query_user（list_type, match_type, sort_order, page）。

争论 描述
列表_类型 要创建的数据列表的类型（见下面列出的常数）。
匹配类型 要查询的UGC项目的类型（见下面列出的常数）。
排序_顺序 数据应该被排序的方式（见下面列出的常数）。
页 要查询的页码。

返回。

查询手柄

例子。

query_handle =steam_ugc_create_query_user(ugc_list_Published, ugc_match_Items, ugc_sortorder_TitleAsc, 1);

上面的代码创建了一个查询请求，并把它的句柄存储在一个变量中，以便将来使用。

返回。UGC
