点_产品

点积是一个表达两个向量之间角度关系的数值，通过将两个向量相乘，然后将结果相加而得到。点积 "这一名称来自于中间的点"-" 衍生出来的（另一个名称 "标量积 "强调了结果的标量性质，而不是矢量性质），通常用来表示这种操作。

实际的数学公式可以这样写。

因此，在二维空间中，向量 a[x1,y1] 和 b[x2,2] 的点积是 x1x2 + y1y2 ，这意味着在GameMaker Studio 2中点积的计算方式是：。

a-b=（x1*x2）+（y1*y2）。

关于点积的奇怪之处在于它与输入向量形成的角度的关系，可以表示为：。

a-b=（a的长度）*（b的长度）*cos(角度)

也就是说，两个向量的点积将等于这些向量之间的角度的余弦，乘以它们各自的长度。这里有一个图像来说明。

现在我们可以从任何两个向量的点积结果中说明一些事情。

  • 如果a和b是垂直的（彼此成90度），点积的结果将是零，因为 cos(ang) 。
  • 如果a和b之间的角度小于90度，点积将是正的（大于零），因为 cos(ang) ，将是正的。
  • 如果a和b之间的角度大于90度，点积将是负的（小于零），因为 cos(ang) 。

那么，这对我们这些做游戏的人来说意味着什么呢？好吧，这种数学关系可以用在很多情况下，但最好的方法是建立一个实际的场景，自己看看发生了什么。最简单的方法之一 最简单的方法之一是为敌人生成一个简单的 "视线 "检查，比如说，在一个平台游戏中，如果玩家在其矢量法线的任何一侧90°以内，敌人就会 "看到 "玩家。

基本上，我们必须从敌人那里获得法线矢量（即：由方向和视距组成）。 然后我们需要得到玩家与敌人的向量。然后我们将得到这些向量的点积，如果结果是正的，玩家就被看到了，如果是负的，他就没有被看到。实际的代码是这样的 在下面的例子中。

语法。

dot_product(x1, y1, x2, y2)

争论 描述
x1 第一个矢量的X坐标。
y1 第一个矢量的Y坐标。
x2 第二个矢量的X坐标。
y2 第二个矢量的y坐标。

返回。

真实的

例子。

var x1, y1, x2, y2;
x1 = lengthdir_x(1, image_angle);
y1 = lengthdir_y(1, image_angle);
x2 = o_Player.x - x;
y2 = o_Player.y - y;
if dot_product(x1, y1, x2, y2) > 0 seen=true otherwise seen=false;

上面的代码用实例图像的角度创建了一个向量，然后把播放器对象 "o_Player "的向量拿到自己身上。最后，它计算这两个向量的点积，如果它大于0，就把变量 "看到的 "为真，如果它等于或小于0，则将其设置为 "假"。
