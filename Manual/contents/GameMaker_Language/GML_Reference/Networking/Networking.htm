联网

GameMaker Studio 2的网络功能是基于套接字的，这是所有平台的行业标准。有了套接字，你就有能力使用简单的GML来创建客户端和服务器，这样你就可以使用客户端/服务器模型来创建一个单人游戏。 甚至可以使用客户端/服务器模型创建一个单人游戏（以后可以轻松添加多人游戏）。

基本上，套接字是一个可以在网络上发送、接收、连接和监听端口的对象。这些函数只处理TCP/IP连接，这也是互联网的基础（更准确地说，是IPv4）。从本质上讲，这可以让你 用一个IP地址（比方说192.168.1.10）连接一个套接字到另一个IP地址（比方说192.168.1.11）上的套接字。一旦连接，这两个套接字可以使用network_send_*函数来回发送数据（比如 network_send_packet()) 以及由它们触发的专用异步事件。这个异步事件将生成一个DS地图，其中包含所收到的数据以及其他细节，你可以在网络异步事件一节中找到完整的细节 你可以在网络异步事件的章节中找到完整的细节。

与其说程序不得不读取和处理进入机器的每一点网络流量，不如说IP地址也处理端口，端口的编号从0到65535。这允许每个程序只获得它感兴趣的数据包，而不是每个程序发送的所有数据。 的数据包，而不是每个程序都发送过的所有数据。这也节省了相当多的CPU时间，因为它严重削减了数据处理，所以用GameMaker Studio 2你也可以 "监听 "端口。因此，一个服务器将创建一个套接字，然后 绑定到一个特定的端口来监听。然后，它将收到来自该套接字的连接、断开连接和数据传输的通知。

数据 "包 "本身是由取自缓冲区的二进制数据组成的。这个缓冲区可以使用GameMaker Studio 2 Buffer函数来创建和操作。

值得注意的是，如果你正在为HTML5目标创建一个项目，那么由于浏览器的限制，你将无法使用服务器的创建功能，你必须为这个平台编写自己的服务器，使用像Electron 或 NodeJS。还需要注意的是，如果你的游戏是跨平台的，并且你打算从运行在移动或桌面设备上的服务器连接到游戏的HTML5实例，那么你的服务器只能使用Web Socket协议与HTML5实例通信 使用Web Socket协议（这是通过常量network_socket_ws实现的--见 network_create_server()获取更多信息）。)这意味着您的服务器可能需要在不同的端口创建两个套接字 在不同的端口上，一个用于连接非 HTML5 目标（Android、Windows 等），一个用于连接 HTML5 目标。
