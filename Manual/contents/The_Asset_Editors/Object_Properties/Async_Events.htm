异步事件

异步事件是指当GameMaker Studio 2收到来自某个外部来源的回调时被触发的事件，它可以是来自网络或运行游戏的设备。基本上。 你告诉GameMaker Studio 2做一些事情--比如加载图片--它会开始做这件事，但同时继续做它要做的其他事情。然后，当请求完成后，一个回调将被发送到GameMaker Studio 2，为该类型的回调定义的任何异步事件将被触发。

请注意，异步事件是为所有 拥有这些事件的实例而触发的，就像键盘事件一样，所以你可以--例如--在一个实例中做一个 http_get() ，但在另一个实例中有异步HTTP事件来处理回调。

注意：你应该知道，由于浏览器的XSS保护，在使用以下任何事件时，从跨域加载资源的请求和尝试会被阻止，并可能出现返回空白结果。

有各种类型的事件与异步事件类别相关，它们都在下面的章节中解释。

注：提到的变量 async_load 在以下章节中提到的 只在这些事件中有效，因为它所指向的DS地图是在事件开始时创建的，然后在结束时再次删除，在所有其他时间，这个变量被重置为-1的值。
