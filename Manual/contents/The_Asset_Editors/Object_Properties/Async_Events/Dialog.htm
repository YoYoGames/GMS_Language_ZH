对话

对话框异步事件只有在得到一个回调的时候才会被触发。 特殊的异步对话函数的回调时才会被触发，比如 get_login_async()(请 请看这个函数中关于如何使用该事件的一个扩展代码示例）。)

这些事件是要求某种类型的用户输入，可以是姓名、登录信息、数字或颜色等等。由于大多数设备不喜欢坐在一个循环中等待回复，它们必须是异步的，GameMaker Studio 2将在这些功能打开对话时继续在后台运行，直到它们得到回复。 在这些功能打开对话时继续在后台运行，直到它们得到所需的用户输入，触发这个事件。这意味着，如果对话在游戏进行中被打开，你应该采取措施来暂停 暂停游戏，直到收到对话的回调。

同样，一个DS地图被返回，ID被保存在特殊变量中 async_load. 这个地图中的值取决于所使用的函数，你应该查阅本手册中每个函数的单独条目以了解更多细节。

注意。异步对话功能 只用于调试和测试，不应该在已发布的游戏中使用。为此，你应该创建一个自定义的用户界面来接收玩家的输入，这样你就可以完全控制对话框的外观和行为。
