源头控制

源控制管理（SCM）是指通过资源库和本地源对项目的子版本或备份进行处理的方法。基本上，一个SCM解决方案是一个独立的软件包 它在你工作时控制维护、改变和比较你的项目版本的所有方面。这对那些在团队中工作的人特别有用，他们需要能够控制谁做什么，而不用担心丢失数据或 这对那些在团队中工作的人特别有用，他们需要控制谁做什么，而不用担心丢失数据或做出可能需要在以后撤销的改变，但个人也可以从这个强大而灵活的系统中受益。

你可以使用各种单片机工具，GameMaker Studio 2包含了一个Git插件，可以直接使用，你不需要安装任何额外的软件包，因为它都是与插件本身捆绑在一起的。下面你可以找到 关于如何设置这个插件和在项目中使用SCM工具的小教程的链接。

当你为任何项目激活了源码控制（详见上面的设置源码控制Git插件一节），GameMaker Studio 2将在IDE顶部显示以下上下文菜单 在IDE的顶部显示以下上下文菜单。

这里我们概述了每个可用的选项（其中大部分在上面的教程部分有更深入的解释）。

  • 创建项目存储库。这个选项允许你在与你的项目文件相同的目录下创建一个本地资源库。
  • 导入项目到存储库。该选项允许你将一个项目导入到外部资源库。
  • 克隆版本库。这个选项将允许你从源头克隆一个版本库到一个新的目标。
  • 提交更改。有了这个选项，你可以在你的项目中对更改的文件进行分期，然后将它们提交到版本库。
  • 推送更改。在做完提交或合并后，这个选项用来推送修改到主版本库。
  • 拉动更改。有了这个选项，你可以通过从主版本库拉取更改的文件来更新本地版本库。
  • 查看历史。这将打开历史窗口，你可以查看项目的所有版本历史，并选择回滚特定的文件路径或整个修订版。
  • 显示冲突。这将打开冲突窗口，列出项目中可能存在的任何冲突文件，允许你通过GameMaker Studio 2IDE或使用特定的合并工具来处理它们。
