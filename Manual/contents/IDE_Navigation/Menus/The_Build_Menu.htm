构建菜单

在构建菜单中，你可以选择构建你的项目，用于测试、调试或作为最终可执行文件。可用的选项是。

  • 运行 - 在选定的目标平台上运行你的当前项目进行测试。关于编译时可用的不同编译选项，请参见编译一节。请注意，这个 命令与在IDE中按下运行按钮 是一样的。
  • 调试 - 用调试器运行你当前的项目进行测试，这样你就可以看到详细的性能信息并调试你可能遇到的任何错误或问题。关于调试的更多信息，请参见调试一节。 注意，这个命令与在IDE中按下Debug按钮 。
  • 清洁 -GameMaker Studio 2保留了一个文件缓存，以帮助在测试时加快编译时间。然而，有时一个 "陈旧 "的缓存会导致你的游戏出现奇怪的和意外的错误。 在这种情况下，你应该做的第一件事就是从这里清理缓存并重新测试。在你为目标平台创建最终的可执行文件之前，清理缓存也是一个好主意。注意，这个命令与按下 Clean Cache按钮 。
  • 创建可执行文件- 这将编译当前项目并为目标平台创建一个可执行包（或多个包）。它将首先打开文件资源管理器 这样你就可以选择你想保存项目的位置，然后编译游戏。关于设置编译目标的更多信息，见编译一节。请注意，这个命令与按下创建可执行文件按钮 与在IDE中按下Create Executable按钮（ ）相同。
  • 显示Remote Worker安装程序- 这将打开一个文件资源管理器窗口，显示Remote Worker工具的安装程序位置（关于Remote Worker的更多信息，请看这里）。 的更多信息）。)
