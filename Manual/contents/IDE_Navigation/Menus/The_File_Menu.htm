文件菜单

文件菜单是你可以选择更改、保存或导入项目以及设置GameMaker Studio 2IDE偏好的地方。可用的选项有

  • 新项目 - 这是在GameMaker Studio 2中从头开始创建一个新项目。它将打开 "保存 "对话框，并提示你选择一个位置来保存新项目。一旦你给项目取了一个名字 和保存位置，你就会被带到一个新的、空白的工作区，开始进行工作。项目文件以 *.yyp 文件扩展名保存。
  • 打开项目- 这将打开一个文件资源管理器，让你选择一个先前创建的 *.yyz 项目文件进行加载。请注意，在导入时，如果该项目是用低于2.3.0的版本创建的。 那么你将得到以下警告信息： 如果你选择继续，那么文件资源管理器将被打开，你可以 给予项目一个更新的名称和保存位置，这样原项目就不会被改写了。一旦项目被转换为新的2.3.0+格式，它就会在IDE中打开，你会得到一条信息，说转换 已经成功。你应该继续检查转换过程中生成的兼容性报告，看看在尝试运行或编译项目之前是否有什么需要修复的地方。请注意，如果你在试图打开一个用 "D "字开头的项目时，重复出现错误 请注意，如果你在试图打开一个用2.3.0以前的版本制作的项目时反复出错，那么你应该联系YoYo Games（使用帮助菜单中的 "报告错误 "选项），提供你试图转换的原始 *.yyp 。
  • 导入项目- 在这里你可以导入以前作为 *.yyz 档案导出的项目（见下面的 "导出项目"），或者--如果你有一个用GameMaker:Studio 1.4 软件制作的项目--你可以选择这个选项来导入该产品系列的 *.gmx 或 *.gmz 文件。与打开项目一样，如果 *.yyz 是用2.3.0更新之前的GameMaker Studio 2版本创建的，那么它将需要转换，因为项目文件格式随着该更新而改变。

注意：要把以前的GameMaker项目导入GameMaker Studio 2，它必须是用1.4版本创建的，即使如此，它也可能无法直接导入。在GameMaker Studio 2和GameMaker 2之间有很多不同的地方。 GameMaker。Studio 1.4之间有许多不同之处，所以你的老项目可能需要一些调整才能工作。请参阅YoYo Games上的文章《将GM:S 1.4游戏移植到GameMaker Studio 2》。 知识库和手册中涉及过时的功能和兼容脚本的部分。

重要的是!导入一个包含字体资源的 *.yyz 项目可能需要你通过删除/重新添加字体来 "重新生成 "字体，或者在字体编辑器中改变字体，然后再设置为 然后再把它放回去。这是因为现有的字体信息在导出时被 *.yyz ，而在以后导入时或在另一台机器上，该项目可能已经没有安装所需的字体（或者可能有不同的同名字体）。 有相同名称的不同字体）。)只有当你看到字体在导入后的渲染效果有问题时，才应该这样做。

  • 最近的项目- 你可以使用这个选项来查看最近使用的项目。选择它将重新打开 "起始页"（如果你已经关闭了它），你可以从那里的列表中选择一个最近使用的 项目，你可以从那里的列表中选择最近使用的项目。
  • 保存项目- 这将把你的游戏保存到你创建项目时定义的相同位置（或者你从哪里打开它）。
  • 将项目另存为- 和上面一样，只是你要指定项目的位置和新名称。
  • 导出项目 - 这将把项目导出为一个单一的 *.yyz 文件。对于向人们发送副本或保存备份非常有用。
  • 首选项 - 选择它将带你到首选项窗口，你可以设置GameMaker Studio 2的外观和感觉。
  • 新的IDE- 当你选择它时，一个新的IDE实例将被打开供你使用（它将在开始页上打开）。
  • 我的账户- 该选项将打开您的默认网络浏览器，并将您带到您的悠悠账户页面。
  • 注销- 选择此选项将使你退出GameMaker Studio 2。你需要用同一个许可证（或任何其他许可证）再次登录，以便以后能够使用该产品。

重要的是!如果你一直在使用GameMaker Studio 2的试用许可证，然后购买了其他许可证（桌面、移动、UWP、Web或控制台），那么你可能需要注销IDE，然后重启GameMaker Studio 2，再重新登录。 再次登录，以确保新的许可证被正确应用（尽管这应该自动发生）。

  • Exit - 退出GameMaker Studio 2到桌面。
