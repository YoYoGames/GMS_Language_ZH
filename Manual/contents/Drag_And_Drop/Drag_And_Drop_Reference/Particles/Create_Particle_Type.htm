创建粒子类型

在你的项目中，你可以有多个不同的粒子类型，每个粒子都有自己的属性，比如方向和颜色等等......这些都可以通过不同的粒子类型动作来设置。然而，你首先需要使用这个动作来创建一个粒子类型，这基本上是分配了一小部分内存来保存所有的粒子细节，这样你就可以在未来的任何时候使用这些粒子。你给一个变量来保存粒子类型的唯一ID值，然后在 所有与这个粒子有关的未来行动中，你可以调用这个变量。你可以选择将这个变量标记为一个临时的局部变量，但这意味着它将在动作脚本或事件结束时被丢弃，这意味着粒子 类型仍然存在，但你却没有办法访问它，所以在使用时必须小心（一般来说，粒子类型永远不需要存储在一个临时的局部变量中，但对于某些事情，比如将粒子绘制到表面。 它可能是必需的）。)

注意：粒子类型一旦创建，就会在游戏过程中一直驻留在系统内存中，因此如果你需要在任何时候重新创建它（例如，如果你调用重启游戏动作）。 那么你就必须销毁当前的粒子类型，否则就会出现内存泄漏。当一个资源被添加到内存中，然后被 "取消引用 "时，就会发生这种情况，这意味着GameMakerStudio 2没有办法再与该资源进行交互。 的互动。这导致内存被无法访问的资源堵塞，这反过来会导致你的游戏出现问题。要销毁不再需要的粒子类型，请使用Destroy Particle Type动作。

你还可以设置粒子的混合模式。默认情况下，它们将以 "普通 "混合模式绘制，因此颜色将不会被绘制在粒子下面的东西所改变。 启用时，它们将使用加法混合模式绘制，这意味着事物的颜色将根据粒子下面绘制的内容而改变。加法混合可以产生非常明亮和 "发光 "的效果，并且是 爆炸、激光、火焰等的理想效果，而普通混合则更适合于雨、烟和碎片等效果。下面的图片显示了关闭（左）和打开（右）加成混合的相同效果。
