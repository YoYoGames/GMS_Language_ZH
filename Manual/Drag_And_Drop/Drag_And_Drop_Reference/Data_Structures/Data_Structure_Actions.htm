数据结构动作库

在游戏中，你经常需要以精确和有序的方式存储信息。例如，你可能需要存储一个人携带的物品清单，或者你可能想存储一个仍然需要访问的地方的网格。现在，你可以使用多个变量 来实现，但如果你想做更复杂的操作，比如对数据进行排序或搜索某个特定的项目，你就需要编写大块的动作块代码，这样执行起来会很慢，难以调试，而且编写起来也很麻烦。

为了弥补这一点，GameMaker Studio 2有一些内置的数据结构，可以通过专业的函数来访问。有四种不同类型的数据结构可以通过拖放（如果你使用 GML), 每一种都有自己的优点和缺点，这取决于你想要存储的信息类型以及你希望以后如何操作它--这四种类型是：。

列表 列表是一种数据结构，在信息被添加时按顺序存储。列表的索引从0开始，每添加一个项目到列表的末尾都会上升，尽管你也可以把数据插入到最后一个位置以外的位置。与Stack数据结构不同（在下面解释），你可以在任何时候从列表的任何索引访问数据。
地图 地图数据结构是一个非常有用的结构，因为它可以让你存储 钥匙和 值对。例如，在你的游戏中，一个角色可以拥有一定数量的不同物品（键），对于每个单独的物品，他可以拥有一定数量的物品（值），比如在RPG游戏中，你可以拥有10个健康药水，5个法力药水和100个金币。地图将这些物品对集中在一起，你可以在地图上添加物品对，也可以使用一些简单的函数搜索某些键所对应的值。地图没有以任何（可识别的）方式进行排序，这意味着要找到某个键，你可能必须遍历整个地图（这很慢），而且也没有办法保持两个相同的键，也不能给一个键分配两个值。
网格 网格是二维的数值阵列，你定义它的宽度和高度。这就设定了网格内的网格 "单元 "总数（宽度和高度相乘），然后这些单元被用来保存不同的数据值，这些数据可以是任何允许的数据类型。这种结构允许你通过给出它的x（列）和y（行）位置来设置和检索网格中的单元值（行和列都从0开始，一直到网格宽度-1和网格高度-1）。当访问网格数据结构时，你应该总是尝试使用整数值来表示单元格的位置，如果你不这样做，所有非整数指数都会被 GameMaker 工作室2会将所有的非整数指数浮动 。如果这不是你所要求的，那么你就需要在传递你希望检查的索引之前，自己事先进行舍入。
堆栈 堆栈数据结构是一个所谓的后进先出（LIFO）结构。你可以把数值推到堆栈上，然后再从堆栈中弹出，最近推到堆栈上的数值是第一个被弹出的（想想看，在一摞硬币中，你添加的每一个硬币都要先被取出，然后才能拿到下面的其他硬币）。当有中断需要处理时，或有递归函数时，甚至在为你的游戏构建一个基本的人工智能时，经常使用堆栈。

从本质上讲，所有数据结构的工作方式都是一样的--你创建一个数据结构并将其索引值存储在一个变量中。然后，在所有可以对其进行操作的后续函数调用中，使用这个索引来引用该数据结构。最后。 一旦你完成了，你就再次销毁该数据结构，将其从内存中删除。你可以根据需要同时使用多个结构，所有结构都可以存储字符串和实值。

注意：与所有的动态资源一样，数据结构会占用内存，所以当不再需要时，应该总是销毁，以防止内存泄漏，这将减慢并最终使你的游戏崩溃。
